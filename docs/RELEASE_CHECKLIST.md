# 发布前检查清单

## 📋 发布资料准备

### 1. 截图准备（必需）

需要准备至少 5 张高质量截图：

#### 截图1：主界面展示（1200x800）
- **内容**：完整的播放器界面，包含视频、控制条、标题
- **场景**：PC 端浏览器全屏
- **要点**：展示播放器的整体设计风格

#### 截图2：分类视频画廊（1200x800）
- **内容**：分类卡片 + 视频网格列表
- **场景**：完整页面视图
- **要点**：突出分类功能和视频展示

#### 截图3：移动端响应式（375x812 iPhone 尺寸）
- **内容**：移动端播放器界面
- **场景**：手机浏览器或开发者工具模拟
- **要点**：展示响应式适配效果

#### 截图4：交互功能展示（1200x800）
- **内容**：控制面板细节、全屏效果等
- **场景**：操作过程截图
- **要点**：展示核心交互功能

#### 截图5：代码示例（1200x800）
- **内容**：使用代码示例 + 运行效果
- **场景**：编辑器 + 浏览器并排
- **要点**：展示易用性

### 截图工具推荐
- **Mac**：Command + Shift + 4（区域截图）
- **Windows**：Win + Shift + S（截图工具）
- **Chrome DevTools**：响应式模式截图
- **专业工具**：Snagit、Xnapper、CleanShot X

### 截图优化
1. 分辨率：至少 1920x1080，导出为 1200x800
2. 格式：PNG（保持清晰度）
3. 文件大小：每张不超过 500KB
4. 压缩工具：TinyPNG (https://tinypng.com/)

### 2. 演示 GIF 制作（推荐）

#### 录制内容
1. **基础操作流程**（10-15秒）
   - 打开页面
   - 点击播放
   - 拖拽进度条
   - 全屏播放

2. **分类切换演示**（10-15秒）
   - 点击分类卡片
   - 自动加载视频
   - 选择视频播放

3. **响应式效果**（10-15秒）
   - 浏览器窗口从大到小
   - 展示布局自适应

#### 录制工具
- **Mac**：Kap (免费), ScreenFlow (付费)
- **Windows**：ScreenToGif (免费), Camtasia (付费)
- **在线工具**：Recordit, Giphy Capture
- **Chrome 扩展**：Loom, Screencastify

#### GIF 优化
- 尺寸：800x600 左右
- 帧率：10-15 FPS
- 时长：不超过 15 秒
- 文件大小：不超过 3MB
- 优化工具：ezgif.com

### 4. Logo 设计（推荐）

#### Logo 要求
- 尺寸：512x512（方形）
- 格式：PNG（透明背景）
- 风格：简洁、现代
- 颜色：与插件主题色（#6EBA44）相关

#### 设计元素建议
- 播放按钮图标
- 视频播放器轮廓
- 音视频波形
- 字母 "IMP"（Immersive Media Player）

#### 设计工具
- **在线工具**：Canva, Figma
- **专业软件**：Adobe Illustrator, Sketch
- **AI 生成**：Midjourney, DALL-E
